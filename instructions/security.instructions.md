# Security Instructions

## セキュリティ方針

### 認証・認可

- 認証は一元管理する
- API Routes には認証ミドルウェアを適用する
- 権限チェックはサービス層で行う

### 入力検証

- すべての外部入力をバリデーションする（Zod 推奨）
- SQL インジェクション、XSS 対策を徹底する
- ファイルアップロードはサイズ・種類を制限する

### 秘密情報管理

- シークレットは環境変数で管理する
- `.env` ファイルは `.gitignore` に含める
- クラウドの Secrets Manager を利用する

### 依存関係

- `npm audit` を定期的に実行する
- 重大な脆弱性は即座に対応する

### データ保護

- 個人情報は最小限の保持にとどめる
- 通信は HTTPS を必須とする
- ログに秘密情報を出力しない

## セキュリティレビュー

レビュー結果は `docs/security/<feature>-security-review.md` に出力する。

### レビュー観点

1. 認証・認可の適切性
2. 入力境界の安全性
3. 秘密情報の管理状態
4. 権限境界の正確性
5. 外部公開面のリスク
6. 依存ライブラリの脆弱性
